An interactive walk-through of the greedy correlated-equilibrium allocation (Algorithm 1). Set the parameters, then step through Phase 1 (greedy selection) and watch Phase 2 (round-robin allocation) and the per-transaction bribe tolerated assemble in real time.

Parameters

Defaults reproduce the paper's example: n=3, k=2, U=[11,10,9,6,4,3]. Phase 1 makes n·k greedy picks, each time choosing the object with the highest marginal utility U⌀N_c; an object is retired once it has been selected by all n proposers. The denominator grows by γ per selection, matching the utility u_i /(1+γ(n_i−1)).

Selected objects are sorted by finalized utility U_f = u ⌀ (N_c−γ) and dealt round-robin, so an object chosen n_i times lands in n_i distinct input lists. This is the allocation every proposer can simulate — the correlation device.

Comparison · naive MCP (mixed NE) vs. the coordination attack

A naive multi-proposer scheme (MCP-style) has no correlation device, so rational proposers play a mixed Nash equilibrium: each independently selects transactions with probability proportional to fee. Collisions make this inefficient — a correlated equilibrium Pareto-dominates it. An adversary exploits that gap: posing as a coordinator, it offers everyone the efficient CE that omits a target, guaranteeing each party at least its MNE payoff (a conditional top-up covers any shortfall). Parties rationally accept, and the target is censored. AUCIL closes the gap by being the correlation device — it already hands parties the efficient CE including the target, so removing it costs the full Input-CR bribe.
